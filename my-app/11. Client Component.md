# Client Components

# React SPA truyền thống (React, Vite, CRA) là 1 client component khổng lồ

Khi lần đầu vào 1 trang web

1.  Trình duyệt **request** đến server và trả về file `index.html` cơ bản (hầu như không chứa html gì nhiều)
2.  Trình duyệt nhận thấy trong file html có link đến file js, css nên là **request lần nữa** đến server để lấy file js,css
3.  Trình duyệt tiến hành chạy code JS để render ra HTML và gắn event vào HTML đó
4.  User thấy và tương tác với web.

Trong trường hợp này, web sẽ trắng cho tới khi bước 3 được hoàn thành

=> Vậy nên mới nói lần đầu truy cập các SPA truyền thống khá lâu, nhưng sau đó thì thao tác hay chuyển trang sẽ rất nhanh
vì js bundle cả app đã có ở client rồi, nếu cần data thì mới request đến server lấy data thôi.

Ở bước 3:

- Lúc nào HTML cũng được JS trình duyệt render ra khi chúng ta truy cập vào web. Cái này gọi là **Dynamic Rendering**

# Client Components trong Next.JS

Dùng client component khi

- Cần tương tác với các hook
- Cần dùng các API từ trình duyệt

Trong Next.js, mặc định tất cả các component được render ra HTML sẵn khi có thể lúc Nextjs build (Static Rendering).
Kể cả Server Component và Client Component

Khi truy cập vào trang web của Nextjs, UI ngay lập tức được Nextjs trả về HTML đã được render sẵn. Sau đó trình duyệt render lại
Client Component 1 lần nữa để đồng bộ với DOM, sự kiện, state, effect

- Client component bị render tối thiểu 2 lần:
  - 1 lần là khi build
  - 1 lần là ở client
- Vì trả về HTML sẵn nên người dùng có thể thấy content ngay lập tức
